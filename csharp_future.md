# Будущее C#

Информация взята из интервью Сергея Теплякова на канале "Организованное программирование".

Microsoft активно используем сножество разных языков программирования:

- C++ это Core, на котором разрабатываются системные компоненты
- C\# - основа Azure (managed platform) и почти весь Backend
- Rust - next gen для системных компонентов
- Go - часть инструментальных средств, в частности транспайлер TypeScript

Далеко не для всех продуктов Microsoft была выполнена нативная миграция в Linux. Так, например, SQLServer для Linux запускается через дополнительный уровень косвенности, а не как нативное приложение.

Open-source проекты и стартапы редко используют платформу .NET, считая, что .NET является **vendor lock**, что в действительность давно не так.

Очень большой объём полного Framework-а .NET связан с очень глубоко проработанной обратной совместимостью решений.

Первоначально C\# был клоном Java и были скопированы правила из дизайна Java. Многие годы Microsoft борется с последствиями этого решения, устраняя фундаментальыне ошибки в дизайне. Так, например, массив структур работает в C\# гораздо лучше, чем массив объектов, т.е. массив объектов следует дизайн-решениям Java, а структуры позволяют обойти эти ограничения.

Portable-сборка решения .NET может быть очень маленькими по размеру, за счёт отказа от совместимости с разными версиями .NET и триммингу.

**record** - это специальный тип для определения неизменяемых классов данных. Его главная цель — упростить создание типов, которые в основном хранят данные и сравниваются по их значениям.

Ключевые особенности:

- Структурное (по значению) сравнение вместо ссылочного
- Неизменяемость (immutability) по умолчанию. Свойства в позиционной записи используют аксессор init, а не set, поэтому после создания объекта их нельзя изменить
- Выражение with для создания модифицированных копий. Поскольку записи неизменяемы, для "изменения" свойств создают новую копию с обновлёнными значениями: `var updated = p1 with { LastName = "Сидоров" };`
- Автоматически сгенерированный ToString()
- Позиционный синтаксис и канонический конструктор

Когда стоит использовать record:

- Для DTO, моделей запросов/ответов, событий предметной области — везде, где тип в основном хранит данные
- Когда важно сравнение по содержимому (например, в коллекциях HashSet<T>, как ключи словаря)
- В сценариях, где полезна неизменяемость и функциональный стиль обновления состояния
