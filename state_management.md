# Advanced State Management в React

Ключевая особенность состоит в том, что в SPA-приложении разным ветках Component Tree может быть нужен доступ к общему состоянию. Предположим, что мы разрабатываем online-магазин и в это магазине у нас есть корневой элемент App, у которого есть две дочерних ветки:

- App -> Shop -> Product
- App -> Header -> CartModal -> Cart

Поскольку изменение корзины товаров осуществляется в Product, а отображение корзины в Cart (т.е. в разных ветках), мы вынуждены хранить состояние на самом верхнем уровне - в App. Это значит, что в обычном подходе нам потребуется транслировать состояние "Cart State" с уровне App через все компоненты до Product и Cart, что приведёт к тому, что все компоненты будут участвовать в передаче состояния до самого нижнего уровня, т.е. все компоненты будут затронуты.

Передача разделяемых данных через множество уровней компонентов (_components layers_) называется **Prop Drilling**. При этом, абсолютному большинству компонентов, эти данные вообще не нужны и не интересны - они просто передают эти данные своих дочерним компонентам.

>И вот именно Prop Drilling является основной проблемой, т.к. это приводит к тому, что компоненты становятся более сцепленными друг с другом, что критически нарушает гибкость изменения приложения и снижает возможность его повторного использования.

## Концепция Component Composition

Идея состоит в том, что если в типовом подходе у нас используется цепочка вложенных компонентов, то для устранения зависимостей мы выносим элемент нижнего уровня, который работает с состоянием максимально близко к App-у, т.е. к хранилищу состояния.

В коде это начинает выглядеть следующим образом - в компоненте "App.js" мы напрямую можем передавать состояние Product-у:

```js
<Shop onAddItemToCard={handleAddItemToCart}>
    {DUMMY_PRODUCTS.map((product) => (
        <li key={product.id}>
            <Product {...product} onAddToCart={onAddItemToCart} />
        </li>
    ))}
</Shop>
```

Т.е. Shop-компонент всё ещё wrapper для Product, но он уже исключен из трансляции состояния.

Однако, этот подход имеет очень ограниченные возможности применения, из-за чего его следует рассматривать как "костыль", применимый в некоторых случаях.

## React's Context API

Задача этого API состоит в возможности совместного использования данных между компонентами.

Идея состоит в том, что создаётся некоторый wrapper (в общем случае, выше компонента App, но это не обязательно) к которому привязывается состояние (например, Cart State). Т.е. хранилищем состояния перестаёт быть App, а передача этого разделяемого состояния перестаёт осуществляться через Props. Те компоненты, которым нужен доступ к разделяемому состоянию получают этот доступ напрямую, без привлечения родительских компонентов в качестве посредников.

Общая рекомендация - создавать артефакты, имеющие отношение к Context-у в папке "store". В этой папке добавляется файл с суффиксом "-context", например: "shopping-cart-context.jsx". В таком файле описывается контекст:

```js
import { creareContext } from 'react';

export const CartContext = creatContext({
    items: []
});
```

После этого нужно обернуть компоненты, для которых контекс имеет смысл в созданный нами контекст. Чаще всего это делается в "App.js". Сначала импортируется определённый контекст:

```js
import { CardContext } from './store/shopping-cart-context.jsx';
```

Затем компоненты верхнего уровня включаются в этот, представленный уже в виде JSX-компонента, контекст:

```js
<CardContext>
    <Header ... />
    <Shop>
        ...
    </Shop>
</CardContext>
```

Следует заметить, что приведённый выше синтаксис прекрасно работает в React 19, но в более старых версиях нужно использовать "CardContext.Provider". Однако при определении wrapper-а нужно определять поле value:

```js
<CartContext.Provider value={{items: []}}>
```

>В действительности, установка параметра value - это полноценная инициализация, а определение полей в CartContext = creatContext() это подсказка для autoCompletion.

### Использование контекста (Consuming the Context)

Чтобы использовать контекст необходимо импортировать контекст в дочернем компоненте-потребителе:

```js
import { CardContext } from './store/shopping-cart-context.jsx';
```

А также загрузить один из двух hook-ов: **useConext**, или **use** (он действительно называется просто use). Хук use более гибкий, его можно, в частности, использовать в условных операторах. Однако use() может быть использован только в React 19.

Пример:

```js
import { useContext } from 'react';
```

Далее необходимо сгенерировать объект доступа к контексту, используя hook и имя контекста:

```js
export default function Cart({item, onUpdateItemQuantity}) {
    const cartCtx = useContext(CartContext);
}
```

Следует заметить, что при вызове useContext() мы может использовать destructuring:

```js
const { items } = useContext(CartContext);
```

После этого мы моем обращаться к полям контекста по имени cartCtx, например:

```js
<div id="cart">
    {cartCtx.items.length === 0 && <p>No items in cart!</p>}
```
